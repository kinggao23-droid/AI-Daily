## 一、今日要闻

### 1. Google 豪掷 400 亿美元加注 Anthropic：基础模型层进入"国家级基建"竞争

[Google](https://www.cnbc.com/2026/04/24/google-to-invest-up-to-40-billion-in-anthropic-as-search-giant-spreads-its-ai-bets) 宣布将向 Anthropic 投资最高 400 亿美元——首期 100 亿落地，剩余 300 亿与里程碑挂钩。加上 Amazon 本周追加的 50 亿（总额可达 250 亿），Anthropic 同时被两大云厂商深度绑定。

**核心技术原理：模型供应链的"双押注"**

云厂商的策略类似投资组合管理：
- AWS、Google 都不希望被某个模型供应商单点绑死
- 同时押注 Anthropic + 自研模型（如 Google 的 Gemini）+ 开源生态（如 Meta Llama）
- 这种"多模型路由"在企业产品里也常见——通过 AI Gateway（如 LiteLLM、Portkey）做模型选择路由，用对的场景调对的模型

**对产品的启示**：
- 基础大模型已成"基础设施竞争"，中小厂商无力跟进——**应用层和垂直场景才是新机会**
- 选型时不应把鸡蛋放一个篮子，**多模型路由能力**应成为产品标配
- 对于 ToB 产品，"模型中立"叙事比"绑定单一最强模型"更有说服力

### 2. DeepSeek-V4 预览版开源：百万字上下文 + Agent 旗舰能力

[深度求索](https://www.cnn.com/2026/04/24/tech/chinas-ai-deepseek-v4-intl-hnk) 正式发布 DeepSeek-V4 预览版并同步开源，分 pro / flash 双版本。在 Agent 能力、世界知识和推理性能上均实现国内与开源领域领先。路透指出 V4 在世界知识测评中**领先所有开源模型，仅次于 Google 顶尖闭源模型**。

**核心技术原理：百万字上下文是怎么做到的**

传统 Transformer 注意力机制的复杂度是 O(n²)——上下文翻倍，计算量翻 4 倍。突破百万字靠几个关键技术：
- **Multi-head Latent Attention (MLA)**：DeepSeek 自研技术，把 Key/Value 压缩到低维潜空间，KV Cache 占用大幅下降
- **Sliding Window + Global Attention 混合**：大部分注意力只看局部窗口，少数关键 token 才做全局关注
- **Mixture-of-Experts (MoE)**：1T 参数但每次推理只激活约 370 亿，算力消耗可控

**对产品的启示**：
- DeepSeek 系列的开源策略让企业**私有化部署的可选项更丰富**——这对金融、政府等数据敏感行业是利好
- 百万字上下文意味着可以处理整本年报、整个代码库、整套合规规则——RAG 的存在意义被部分削弱
- 但**长上下文 ≠ 替代 RAG**：成本和延迟仍是瓶颈，长上下文适合"一次性深度分析"，RAG 适合"高频动态查询"

### 3. OpenAI 发布 GPT-5.5：进入 "compute-powered economy" 时代

OpenAI 联合创始人 Greg Brockman 在周四发布会宣布 GPT-5.5 正式上线 API，并表示"我们正在进入一个 compute-powered economy"。OpenAI 同步发布了 [新版 prompting 指南](https://platform.openai.com/docs/guides/prompt-engineering)，因为 GPT-5.5 对 prompt 范式做了较大调整。

**核心技术原理："Compute-powered economy" 是什么意思**

Brockman 这句话有特定含义：
- 早期 AI 价值在"训练"阶段——谁数据多、参数大、训练算力足，谁就赢
- 新阶段价值在"推理"阶段——同一个模型，给它更多"思考时间"（test-time compute），输出质量显著提升
- 典型例子：OpenAI o1 / o3 系列，DeepSeek R1，本质都是"让模型先打草稿再回答"

**对产品的启示**：
- 模型升级后**prompt 范式可能完全不同**——产品里的 prompt 模板需要定期回归测试，不能"上线一次用半年"
- 模型选型时不能只看能力榜单，要看**你的实际 prompt 在新模型上能不能稳定迁移**
- 推理时间变长意味着单次成本上升，**批处理 + 缓存**对成本控制更关键

### 4. Glean 完成 1.5 亿美元 F 轮，估值 72 亿美元：企业 AI Agent 赛道真正的赢家

[Glean](https://www.glean.com/) 完成 1.5 亿美元 F 轮融资，估值 72 亿美元，继续扩张企业 AI Agent 能力（组织知识 + 上下文问答）。**这本质上是一家"企业版 ChatGPT + Notion AI"的公司**。

**核心技术原理：通用大模型 + 企业知识 = 最被买单的产品形态**

Glean 的产品逻辑：
- 不自研基础模型——直接用 OpenAI / Anthropic
- 重投入"知识连接器"：把企业内部 100+ 个 SaaS（Slack、Jira、Confluence、Notion、Salesforce…）的知识统一索引
- 重投入"权限对齐"：每个用户问问题时，只能看到自己有权限的数据
- 在通用大模型上加一层"企业上下文"——这才是企业愿意付费的核心

**对产品的启示**：
- 对国内做 LLMOps 平台的厂商，Glean 的成功验证了"**LLMOps + 知识工程**"组合的市场价值
- 客户付费的不是"AI 模型"，而是"AI + 我的知识资产"——LLMOps 的产品故事可以更靠近这个角度
- 权限管理（行级、字段级）是金融/政府客户的核心刚需，但 LLM 的权限对齐至今仍是行业难题

### 5. 一篇值得看的论文：YC-Bench

**《YC-Bench: Benchmarking AI Agents for Long-Term Planning and Consistent Execution》**（arXiv 2604.01212）

专门评测 Agent 的**长周期规划能力 + 执行一致性**——这恰好是企业落地 Agent 最大的隐痛。

**核心技术原理：为什么 Agent 评估这么难**

传统 NLP 任务有标准答案（翻译质量、分类准确率），但 Agent 评估有几个独特挑战：
- **路径多样**：完成一个任务有多条合理路径，无法用唯一标准答案打分
- **时间跨度长**：一个任务可能跑 5-50 步，错误会累积放大
- **环境动态**：第二次运行时外部状态可能已变（数据、文件、API 响应）

YC-Bench 的做法是**分阶段评估**——把长任务拆解成阶段目标，每个阶段独立检查。这个范式已经成为 Agent 评估的主流。

**对产品的启示**：
- 做 Agent 类产品，**评估集要分阶段设计**，不能只看最终结果
- "AI 准确率 95%" 这种笼统指标在 Agent 场景毫无意义，要拆到每个工具调用、每步决策
- 这套方法论可以直接用于企业 Agent 上线后的常态化监控

## 四、趋势洞察与行动建议

### 趋势一：基础模型层进入"少数玩家 + 巨额投资"格局
Anthropic 一周内拿到 Amazon 50 亿 + Google 400 亿，OpenAI 估值 8520 亿——基础模型已不是创业可玩的赛道。**对产品经理的建议**：把焦点彻底放在应用层，特别是**垂直行业 + 数据壁垒**的产品。

### 趋势二：开源能力差距持续收窄
DeepSeek V4 在世界知识上已仅次于 Google 闭源模型；Qwen3.6 开源；Coze 开源全套。**对产品经理的建议**：对私有化部署、数据敏感行业（金融、政府、医疗），**开源 + 私有化** 路线的可行性大幅提升，应纳入选型评估。

### 趋势三：AgentOps 成为新蓝海
LangSmith 成本视图升级、Coze 开源 Loop、InsightFinder 融资 1500 万——**Agent 上线后的治理（监控、评估、归因、成本）**成为新付费点。**对产品经理的建议**：你的产品如果还没有 AgentOps 模块，现在是入场窗口期。

### 趋势四：评估方法论的范式转变
从"绝对打分"转向"成对比较 (Pairwise)"+"分阶段评估"+"定时跑评估"。**对产品经理的建议**：评估能力是产品差异化的关键，不要只把它当作开发期工具，要把它做成**生产期持续运行的服务**。
