## 一、TL;DR（一页版结论）

1. 2025 下半年到 2026 年，ToB 大模型平台已经从「卖 Token / 卖算力」升级为「卖 Agent 工厂 + 卖治理底座」。所有头部厂商的新版本都围绕同一条主线：**Agent 全生命周期（Build → Evaluate → Deploy → Observe → Govern）**。
2. 国外阵营（AWS Bedrock AgentCore、Azure Microsoft Foundry、Google Vertex AI Agent Builder、Databricks Mosaic AI）主攻「**身份 + 治理 + 评测**」三件套，把 Agent 当「数字员工」来配 IAM。
3. 国内阵营（阿里百炼、火山方舟、百度千帆、腾讯 ADP/TI、智谱 BigModel、华为 ModelArts Studio / Versatile）主攻「**MCP 生态 + Agentic RAG + 行业 Agent 模板**」，更强调 0 代码低代码落地速度。
4. 最明显的「颠覆性」替代：
   - 过去靠**人堆流程 + RPA** 的后台作业 → 现在用 **MCP+Agent 网关** 一键替换（如 AWS AgentCore Gateway、百炼 MCP 统一鉴权）。
   - 过去靠**季度人工评测报告**保障模型质量 → 现在用 **Evaluations + Tracing + Guardrail** 在线持续评测（AgentCore Evaluations GA、Foundry Observability GA）。
   - 过去的**自研网关 + Prompt 仓库 + 自建监控**三件套 → 现在被 **LangSmith / Langfuse** 这类 LLMOps 平台统一吃掉。
5. 推荐选型心法：**国内 1 家云厂商 + 1 家独立 LLMOps（如 Langfuse 自建）+ 国外 1 家用于做海外合规镜像站**，同时把 MCP 作为强制接入标准，避免被单一平台绑死。

## 四、新功能「颠覆」清单（用新平台可以替掉什么旧做法）

以下每一条都对应「以前你要做什么 → 现在一个按钮 / 一次配置 → 节约多少」。这块是这次简报给决策层最直接的卖点。

### 4.1 颠覆「内部 API 一个一个接给 Agent」→ MCP Gateway 一键接入

- **旧做法**：每接入一个内部系统（HR/CRM/ERP），AI 中台都要写一个 Function Tool，维护成本爆炸，权限口径不统一。
- **新做法**：
  - AWS **AgentCore Gateway**：把现有 REST / Lambda 一键转成 MCP 工具，并自动管入口鉴权。
  - 阿里百炼 **MCP 统一鉴权 + 计量计费**，所有 MCP 在 Agent 和 Workflow 里一键调用。
  - Databricks **Unity Catalog + MCP**：跨组织共享 MCP 工具目录，自带审计。
- **颠覆点**：**"AI 网关 + 工具注册中心"** 这一层不用再自研了，平均可省 3-6 人月 / 年的中台维护。

### 4.2 颠覆「季度离线评测 + Excel 对比」→ 在线持续 Evaluations

- **旧做法**：新模型上线前人工跑一次榜单，上线后靠客服投诉发现问题。
- **新做法**：
  - AWS AgentCore **Evaluations GA**（2026-03）：13 个内建评测器（正确性、帮助性、安全性、Tool 使用、目标达成率），阈值告警。
  - Azure Foundry **Evaluations / Tracing / Monitoring GA**：Groundedness / Safety / Risk 在线评估。
  - LangSmith **Insights Agent**：按计划自动找失败模式。
- **颠覆点**：**质量保障从季度工程变成日常监控指标**，和 APM 一样日常化，可以和 SLO 绑定。

### 4.3 颠覆「Agent 共用一个服务账号」→ Agent 自己的身份证

- **旧做法**：所有 Agent 共用一个 System Token，一旦泄露全线沦陷；事后无法定位是谁干的。
- **新做法**：
  - Microsoft **Entra Agent ID**：每个 Agent 一个企业身份，能受 Conditional Access / Identity Protection 管控。
  - Google Vertex：Agent 是一等 IAM Principal，可以设细粒度策略和资源边界。
  - Databricks：所有 Agent / MCP / 模型 / 工具都在 UC 里有 Owner 和完整血缘。
- **颠覆点**：**让 Agent 符合零信任 / SOX / 等保**，把"AI 能做的事"和"合规能审的事"对齐。

### 4.4 颠覆「人工搬运 + RPA 脚本」→ 长程自主 Agent

- **旧做法**：跨系统流程用 RPA 搬运，一改表单就崩。
- **新做法**：
  - 智谱 **AutoGLM / GLM-PC**：50+ 步长程操作自主执行。
  - 华为 **Versatile**：多轮数据处理 + 跨系统流程调度预置模板。
  - 百度千帆 **AgenticRAG + 工具调度 95%+**。
- **颠覆点**：**"RPA + 工单 + 人"三件套可以被一个业务 Agent 替换**，尤其适合运维、合规审查、对账、售后答复。

### 4.5 颠覆「自建 Prompt 仓库 + 监控 + 红蓝对抗」→ LLMOps 一揽子

- **旧做法**：Prompt 放 Git，监控用 Grafana，评测自研，三者割裂。
- **新做法**：
  - **LangSmith**：Observability + Prompt + Dataset + Playground + CLI。
  - **Langfuse**：同上 + 全开源 + 支持私有化（被 ClickHouse 收购后底座更稳）。
- **颠覆点**：**LLMOps 不需要自建**，一个平台吃掉原本三个工具。

### 4.6 颠覆「AI 应用做出来没人用」→ Agent Store + 内生变现

- **旧做法**：做完 Agent 放在内网门户吃灰。
- **新做法**：
  - 阿里百炼 **Agent Store + AI 打赏**：Agent 可被 C 端直接打赏（接支付宝）。
  - 腾讯 **ADP 插件 150+**：直接发到企业微信 / 公众号场景。
- **颠覆点**：**Agent 不再是成本中心，而是可度量 ARR 的数字员工 / 数字产品**。

| LangSmith | "LLMOps 事实标准之一" | Insights Agent + NVIDIA 合作 | SaaS 为主，国内访问不便 |
| Langfuse | "可私有化的开源 LLMOps" | 开源 + ClickHouse 加持 | 自建运维成本需评估 |

## 六、机会与威胁

### 机会（对「自建平台 / 中台」的团队）

1. **工具治理真空**：大多数国内厂商的 MCP 统一鉴权 / 审计还不成熟 → 有机会自己打一层薄的「企业 MCP 治理层」。
2. **行业 Harness 稀缺**：国内厂商的评测能力普遍 ★★，而金融 / 医疗 / 政务有强合规的评测需求 → 有机会做行业定制的 Eval 套件。
3. **跨云 / 跨模型路由**：华为 Versatile 的多模型路由是个好引子，但国内其他平台还没跟上，企业自建路由层可同时压国内 + 海外模型。

### 威胁（对任何采购方）

1. **Lock-in 风险**：Azure Entra Agent ID、AWS AgentCore Policy、Databricks UC 都在绑定自己的身份 / 治理体系；一旦用上切换成本很高。
2. **价格结构即将变化**：Google Vertex Agent Engine 自 2026-01-28 开始按多个子服务计费；AWS Evaluations 按评测次数计费——PoC 阶段看起来便宜，生产阶段会被"评测费 / Tracing 费 / 网关费"叠加放大。
3. **安全赛道爆发**：Zenity / Model Armor / Agent Governance Toolkit 都是 2025-2026 年集中出现的，意味着**"Agent 越界"已被主流厂商认定为头号生产风险**，PoC 阶段必须提前接入。

## 七、给采购 / 自研团队的行动建议（Strategic Implications）

1. **先定一组强制要求，再选平台**（建议直接纳入 RFP）：
   - 必须原生支持 **MCP**。
   - 必须提供 **Agent 独立身份**（或至少可对接 IAM）。
   - 必须提供 **在线 Evaluations + Tracing（OpenTelemetry）**。
   - 必须支持 **KMS / VPC / 审计日志导出**。
   - 必须提供 **Prompt / Dataset 版本管理** 能力。
2. **分层选型**（推荐组合，避免单点依赖）：
   - **国内主平台**：阿里百炼 或 火山方舟 或 百度千帆（看现有云底座）。
   - **国外镜像 / 海外业务**：Azure Foundry 或 AWS AgentCore。
   - **LLMOps 观测层**：Langfuse（自建）或 LangSmith（SaaS）。
   - **治理层**：如果选 Azure，直接用 Entra Agent ID + Agent Governance Toolkit；其它云则自建一层「Agent 中心」，对齐零信任。
3. **Harness 先行**：PoC 立项时同步立一条「评测流水线」子项目——否则三个月后 Agent 上线谁都不敢为质量背书。
4. **风险缓释**：
   - 把 Prompt / Dataset / 工具 Schema **三样资产标准化** 存到企业自己的仓库（不要仅存在平台里）。
   - 签合同时把 **评测 / Tracing / 网关** 费率写进 TCO，不仅看 Token 单价。
   - 关注各平台对 **MCP、OpenTelemetry、OpenInference** 这三个开放标准的跟进速度，这是判断厂商「开放度」的三根指针。
